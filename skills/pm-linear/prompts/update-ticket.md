# Update Ticket

## Purpose
Modify an existing Linear ticket with new information, status changes, or refinements.

## Workflow

1. **Identify Ticket**
   - Get ticket ID from user or context
   - Query Linear MCP for current ticket state

2. **Determine Update Type**
   - Status change (todo → in progress → done)
   - Priority adjustment
   - Description refinement
   - Adding context or acceptance criteria
   - Linking related tickets

3. **Make Update**
   - Use Linear MCP to update ticket
   - Add comment explaining change if significant

4. **Confirm**
   - Show user the updated ticket
   - Verify changes are correct
