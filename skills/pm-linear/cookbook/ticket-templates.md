# Ticket Templates

## Bug Template

```markdown
## Description
[Clear description of the bug]

## Steps to Reproduce
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Expected Behavior
[What should happen]

## Actual Behavior
[What actually happens]

## Environment
- Browser/OS: [if relevant]
- Version: [app version]
- User type: [if relevant]

## Screenshots/Logs
[Attach if available]

## Possible Cause
[Technical hypothesis if known]
```

## Feature Template

```markdown
## Goal
[What problem does this solve? Who benefits?]

## Proposed Solution
[High-level description of the feature]

## Acceptance Criteria
- [ ] [Criterion 1]
- [ ] [Criterion 2]
- [ ] [Criterion 3]

## Out of Scope
[What this feature explicitly does NOT include]

## Technical Considerations
[Architecture notes, dependencies, risks]

## Design
[Link to designs or describe UI if relevant]
```

## Chore Template

```markdown
## Task
[What needs to be done]

## Why
[Why is this maintenance/chore needed]

## Scope
- [ ] [Subtask 1]
- [ ] [Subtask 2]

## Done When
[Definition of done for this chore]
```

- [ ] [Criterion 1]
